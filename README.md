# celery

## 📁 Celery Salary Demo: Асинхронная обработка задач с Celery
Celery Salary Demo — это демонстрационный проект, показывающий возможности библиотеки Celery для асинхронной обработки задач на примере расчета зарплат. Вместо блокирующих операций вы получаете фоновое выполнение, очереди задач, обработку ошибок с повторами и мониторинг в реальном времени.

Проект создан для наглядной демонстрации возможностей Celery: асинхронное выполнение, цепочки задач, группы, обработка ошибок, генерация PDF и мониторинг через Flower.

## 💡 Моё мнение о библиотеке Celery
Celery — это стандарт де-факто для асинхронной обработки задач в Python. Вместо того чтобы заставлять пользователя ждать, пока сгенерируется PDF или отправятся письма, вы просто отправляете задачу в очередь и продолжаете работу.

Библиотека проста в использовании — достаточно добавить @app.task к функции и вызвать её через .delay(). Под капотом — Redis или RabbitMQ, которые хранят очередь, и воркеры, которые выполняют задачи. Масштабируется просто: добавил больше воркеров — и нагрузка распределяется.

Для production-проектов Celery незаменим. Он справляется с миллионами задач, поддерживает расписания (Celery Beat), мониторинг (Flower) и легко интегрируется с Django, Flask и FastAPI.

## Установка

1.Клонируйте репозиторий:
```
git clone <your-repo-url>
cd celery-salary-demo
```

2. Создайте виртуальное окружение (рекомендуется):
```
python -m venv .venv
# Linux/macOS
source .venv/bin/activate
# Windows
.venv\Scripts\activate
```

3. Установите зависимости:
```
pip install -r requirements.txt
```

4. Запустите Redis (брокер сообщений):
```
# Локальная установка
redis-server

# Или через Docker
docker run -d -p 6379:6379 redis:7-alpine
```

## 🔧 Структура проекта
```
celery-salary-demo/
├── tasks.py              # Celery задачи (расчет зарплаты, уведомления, PDF)
├── main.py               # Демонстрация всех возможностей (5 работающих примеров)
├── config.py             # Настройки (Redis, налог 13%)
├── periodic_tasks.py     # Периодические задачи для Celery Beat
├── monitoring.py         # Запуск Flower для веб-мониторинга
├── requirements.txt      # Все зависимости
├── README.md             # Документация
└── .gitignore            # Исключения
```

## 📦 Назначение файлов
## 📦 Структура проекта

| Файл | Что делает |
|------|------------|
| `tasks.py` | Главный файл с Celery задачами: расчет зарплаты, отправка уведомлений, генерация PDF, обработка ошибок |
| `main.py` | Демонстрация: базовое использование, цепочки, группы, ошибки, PDF |

| `periodic_tasks.py` | Периодические задачи: ежемесячный расчет зарплаты, ежедневная проверка посещаемости |
| `monitoring.py` | Запуск Flower — веб-интерфейса для мониторинга задач |
| `requirements.txt` | Зависимости: celery, redis, reportlab, flower |
| `.gitignore` | Исключает `__pycache__/`, `.env`, `*.pdf`, `venv/` и т.д. |
| `README.md` | Документация проекта |
